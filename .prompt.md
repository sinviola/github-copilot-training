---
title: Generate tests
summary: Generate pytest unit and integration tests for selected FastAPI code following repository conventions.
---

Use the current selection or the surrounding FastAPI function to generate tests for this repository.

Rules:
- Create unit tests in `tests/unit/` and integration tests in `tests/integration/`.
- Use `pytest` and `pytest-asyncio` for async functions.
- Use the shared fixtures defined in `tests/conftest.py`.
- Name unit test files `test_<module>.py` and functions `test_<target>_<expected_behavior>()`.
- Mark integration tests with `@pytest.mark.integration` and use `AsyncClient` for HTTP endpoint calls.
- Validate API responses against Pydantic models from `app/main.py`.
- Keep tests fast, deterministic, and isolated.
- Do not create files outside the `tests/` directory.
- Do not modify production code unless absolutely necessary for testability.

Task:
1. Generate unit tests for the selected function or class.
2. Generate integration tests for the selected endpoint if applicable.
3. Use existing application models and endpoint routes from `app/main.py`.
4. Place tests in the appropriate folder using the repository's test structure.

Example command: `/generate-tests` or use this prompt when selecting a function in the editor.
